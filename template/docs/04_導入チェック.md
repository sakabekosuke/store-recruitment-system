# 導入チェック

## Googleフォーム

- [ ] Googleフォームをコピーした
- [ ] フォーム名を変更した
- [ ] 回答先スプレッドシートを設定した

---

## Googleスプレッドシート

- [ ] スプレッドシートをコピーした
- [ ] ファイル名を変更した
- [ ] 応募管理シートへ変更した
- [ ] シート構成を確認した

---

## Apps Script

- [ ] 店舗番号を設定した
- [ ] 通知先メールアドレスを設定した
- [ ] onFormSubmitトリガーを設定した
- [ ] Googleアカウントの権限承認を完了した

---

# テスト応募

Googleフォームから

テスト応募を1件送信してください。

以下を確認します。

- [ ] 応募管理へ登録された
- [ ] 採用管理へ追加された
- [ ] ダッシュボードへ反映された
- [ ] 通知メールが届いた

---

# 採用結果テスト

採用管理シートで

**採用**

へ変更してください。

以下を確認します。

- [ ] 応募管理へ採用結果が保存された
- [ ] 更新日時が保存された
- [ ] 採用管理シートから削除された

同様に

**不採用**

**辞退**

も正常に動作することを確認してください。

---

# トラブルがあった場合

以下を確認してください。

- 店舗番号は正しいか
- 通知先メールアドレスは正しいか
- onFormSubmitトリガーは設定されているか
- Googleフォームの回答先は正しいか
- 応募管理シート名になっているか

詳しくは

**05_トラブルシューティング**

をご確認ください。

## 💡 ワンポイント

Template版では、

Googleフォーム・Googleスプレッドシート・Apps Scriptは

あらかじめ設定されています。

通常、店舗ごとに設定する内容は

- Googleフォームの回答先設定
- 設定シートの店舗番号
- 通知先メールアドレス
- onFormSubmitトリガー

のみです。

それ以外の設定やコードは変更せず、そのまま利用することをおすすめします。
