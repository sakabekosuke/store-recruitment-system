# ⚙️ 03_Apps Script設定

🟩🟩🟩⬜　**進捗：STEP 3 / 4**

---

# はじめに

この章では、

Google Apps Scriptの初期設定を行います。

Template版では、

Apps Scriptのコードはあらかじめ設定されています。

利用者がコードを入力したり編集したりする必要はありません。

**所要時間：約5分**

---

# この章で行うこと

- Apps Scriptを開く
- 店舗番号を設定する
- 通知先メールアドレスを設定する
- トリガーを設定する
- 動作確認を行う

---

# 手順① Apps Scriptを開く

コピーしたGoogleスプレッドシートを開きます。

↓

**拡張機能**

↓

**Apps Script**

をクリックしてください。

以下のファイルが表示されていれば正常です。

- 00_config.gs
- 01_onFormSubmit.gs
- 02_onEdit.gs

---

# 手順② 店舗番号を設定する

スプレッドシートへ戻ります。

↓

**設定**

シートを開きます。

↓

店舗番号を選択してください。

店舗番号を選択すると、

ダッシュボードの店舗名が自動で切り替わります。

---

# 手順③ 通知先メールアドレスを設定する

再度Apps Scriptを開きます。

↓

**00_config.gs**

を開きます。

以下の設定を確認してください。

```javascript
const MAIL = {

  ENABLE: true,

  TO: [

    "store@example.com"

  ]

};
```

通知先メールアドレスを

各店舗で利用するメールアドレスへ変更してください。

複数人へ通知する場合は、

1行につき1件追加してください。

---

# 手順④ トリガーを設定する

Apps Script左側

**トリガー**

をクリックします。

↓

**トリガーを追加**

をクリックしてください。

以下の内容で設定します。

|項目|設定内容|
|---|---|
|実行する関数|onFormSubmit|
|デプロイ|Head|
|イベントのソース|スプレッドシートから|
|イベントの種類|フォーム送信時|

設定後、

**保存**

をクリックしてください。

---

# 手順⑤ 権限を承認する

初回のみ、

Googleアカウントの権限承認が表示されます。

画面の案内に従って

**許可**

してください。

---

# 動作確認

Googleフォームから

テスト応募を1件送信してください。

以下を確認します。

- 応募管理へ登録される
- 採用管理へ登録される
- ステータスが「未対応」になる
- ダッシュボードへ反映される
- 通知メールが届く

---

# よくある間違い

### 通知メールが届かない

00_config.gs の

**MAIL.TO**

へ正しいメールアドレスが設定されているか確認してください。

---

### 採用管理へ追加されない

onFormSubmit のトリガー設定を確認してください。

---

### トリガーを設定していない

Template版でも、

トリガーはコピーされません。

必ず

**onFormSubmit**

のトリガーを追加してください。

---

### 権限承認をしていない

初回のみ

Googleアカウントの権限承認が必要です。

承認後、

再度テスト応募を実施してください。
