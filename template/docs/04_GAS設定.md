# 04_GAS設定

🟩🟩🟩🟩⬜　**進捗：STEP 4 / 5**

---

## はじめに

この章では、Google Apps Script（GAS）の設定を行います。

**所要時間：約10分**

---

## ご安心ください

ここから唯一「プログラム」を扱う作業になります。

ただし、

**コードを書く必要はありません。**

テンプレートのコードを**コピー＆ペーストするだけ**で設定できます。

専門知識は不要です。

---

# この章で行うこと

- Apps Scriptを開く
- 必要なファイルを作成する
- テンプレートコードを貼り付ける
- トリガーを設定する
- 動作確認を行う

---

# 作業の流れ

```
Apps Scriptを開く
        │
        ▼
3つのファイルを作成
        │
        ▼
コードを貼り付ける
        │
        ▼
保存
        │
        ▼
トリガー設定
        │
        ▼
動作確認
```

---

# 手順① Apps Scriptを開く

作成したスプレッドシートの、

**拡張機能 → Apps Script**

をクリックしてください。

新しいApps Scriptプロジェクトが開きます。

---

# 手順② デフォルトファイルを削除

初期状態では

```
Code.gs
```

が作成されています。

このファイルは使用しないため削除してください。

---

# 手順③ ファイルを作成

以下の3ファイルを作成します。

|ファイル|役割|
|---|---|
|00_config.gs|設定ファイル|
|01_onFormSubmit.gs|フォーム送信時の処理|
|02_onEdit.gs|ステータス変更処理|

---

# 手順④ コードを貼り付ける

テンプレートの

```
apps-script
```

フォルダ内にある各ファイルのコードを、

同じ名前のApps Scriptファイルへ貼り付けます。

貼り付け後は保存してください。

---

# 手順⑤ トリガーを設定

左メニュー

**トリガー**

を開きます。

以下のトリガーを追加してください。

|関数|イベント|
|---|---|
|onFormSubmit|フォーム送信時|

※ Template版ではこのトリガーのみ設定します。

---

# 手順⑥ 動作確認

Googleフォームからテスト応募を送信してください。

以下を確認します。

- 応募データへ登録される
- 採用管理へ自動追加される
- ダッシュボードが更新される

以上が確認できれば設定完了です。

---

# よくあるトラブル

### コードが保存できない

→ 波括弧 `{}` や `}` が不足・重複していないか確認してください。

---

### トリガーが動かない

→ 関数名が

```
onFormSubmit
```

になっているか確認してください。

---

### データが反映されない

→ `00_config.gs` のシート名設定を確認してください。

## 💡 ワンポイント

Template版では、店舗ごとの設定変更は **00_config.gs** のみ編集すれば対応できます。

運用中にシート名やステータスを変更する場合も、他のファイルを編集する必要はありません。
